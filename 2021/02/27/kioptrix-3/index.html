Kioptrix Level 3 Writeup

Number three in the Kioptrix series. You can find the VM here Kioptrix Level 3

Discovery

netdiscover

Running netdiscover on the eth1 network interface we see the target running on 192.168.56.6.

Enumeration

nmap

The nmap scan reveals port 80 open running an Apache web server and port 22 running an OpenSSH service. I start a dirb scan on the webserver and visit the site in the browser.

Opening the webapge in firefox reveals Ligoat Security. Searching around the blog and login I find nothing too revealing.

dirb

Dirb reveals a few directories, but /gallery and /phpmyadmin seem to be the most interesting.

Vising /gallery it seems the page is broken, so I open the page source.

The page source reveals a “kioptrix3.com” url that seems to be a potential virtual host.

Opening /etc/hosts and adding the virutal host “kioptrix3.com”, I go back to the browser to switch the ip for the virutal host and refresh to find the page is now fixed.

Searching around the page I find a url query that could be vulnerable.

After adding a quotation to the end of the id param, we do see the site respond with an sql error verifying it is indeed vulnerable to injection.

Starting up sqlmap to automate the process I find the database gallery and a table inside it called dev_accounts.

sqlmap

SQLMap not only dumps the usernames and password hashes but it also cracks them for us.

Revealing two users creds:

  • dreg:Mast3r
  • loneferret:starwars

I ssh into both users, but loneferret seems to have more revealing files.

Catting the CompanyPolicy.README I see a command “sudo ht” that is used by the company to edit and create files.

To fix the terminal I just export the environment variable TERM to be xterm. Now I’m able to run the “sudo ht” command and open the interface.

The application is navigated by using the Function keys with the correct corresponding number. I open the file search feature by hitting F3.

Searching for the /etc/sudoers file and hitting enter I’m presented with the sudoers file.

I see my current user loneferret has sudo privileges to run /bin/bash.

Exiting the “sudo ht” application I run “sudo /bin/bash” and I’m now root. I finish by changing directories to /root and catting the Congrats.txt file to complete the box.
